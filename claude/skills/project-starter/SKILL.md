# Project Starter Skill

## Purpose
This skill enables Claude Code to bootstrap new projects by selecting and integrating skills from the custom-claude-skills repository, setting up proper project structure, and generating comprehensive documentation.

## Repository Reference
- **Skills Repository**: https://github.com/qian-yu-db/custom-claude-skills
- **Available Skills**: 13 specialized skills across Databricks, LangGraph, Python, Planning, and general domains

## Skill Catalog

### Databricks Platform Skills (4)
1. **databricks-asset-bundle** - Generate DAB configurations with serverless compute, modular structure, and multiple input formats (text, Mermaid diagrams, workflow images)
2. **databricks-local-notebook** - Create notebooks with local IDE development via Databricks Connect
3. **databricks-agent-deploy2app** - Deploy AI agents to Databricks Apps
4. **databricks-agent-deploy-model-serving-dab** - Deploy agents to Model Serving using DABs with serverless compute

### LangGraph Skills (4)

2. **langgraph-unstructured-tool-agent** - Build RAG agents with Vector Search (4 patterns: simple, tool-calling, multi-hop, self-query)
3. **langgraph-multi-agent-supervisor** - Build multi-agent systems with supervisor orchestration
4. **langgraph-mcp-tool-calling-agent** - Build agents with Model Context Protocol (MCP) tools

### Python Skills (2)
1. **pytest-test-creator** - Auto-generate comprehensive unit tests with coverage reports
2. **python-code-formatter** - Format code (blackbricks for Databricks, black+isort for regular Python)

### Planning & Visualization Skills (1)
1. **mermaid-diagrams-creator** - Create clean Mermaid diagrams (flowcharts, sequence, class, ER, state diagrams) with automatic PNG/SVG/PDF generation

### General Skills (2)

2. **battle-card-creator** - Automate competitive battle card creation

## Project Structure
When initializing a project, create this structure:

```
project-name/
├── .claude/
│   ├── skills-repo/          # Git submodule of custom-claude-skills
│   ├── skills/                # Symlinks to selected skills
│   │   ├── skill-1/
│   │   └── skill-2/
│   └── project-context.md     # Generated context for this project
├── docs/
│   ├── PROJECT_PLAN.md        # Generated comprehensive plan
│   ├── REQUIREMENTS.md        # Generated requirements
│   ├── ARCHITECTURE.md        # Generated architecture (if applicable)
│   └── SETUP.md              # Setup instructions
├── src/                       # Source code
├── tests/                     # Test files
├── notebooks/                 # Databricks notebooks (if using Databricks skills)
├── configs/                   # Configuration files
├── .python-version            # Python version for uv
├── pyproject.toml             # Project metadata and dependencies (uv managed)
├── uv.lock                    # Locked dependencies (auto-generated by uv)
├── .gitignore
└── README.md
```

**Note**: The project uses **uv** for fast, reliable Python package management instead of pip/venv.

## Initialization Workflow

### Step 1: Understand User Requirements
When user requests a new project, gather:
- **Project name** and description
- **Primary goal** (e.g., build RAG agent, create Databricks app, generate documentation)
- **Skills needed** (based on project type, suggest relevant skills)
- **Tech stack** preferences

### Step 2: Skill Selection Logic
Based on project type, recommend skill combinations:

**For RAG/Agent Projects:**
- langgraph-unstructured-tool-agent (RAG with Vector Search)
- databricks-agent-deploy2app (deployment to Apps)
- pytest-test-creator (testing)
- mermaid-diagrams-creator (architecture visualization)

**For Multi-Agent Systems:**

- langgraph-multi-agent-supervisor (orchestration)

**For Databricks Workflow Development:**
- mermaid-diagrams-creator (design workflow visually)
- databricks-asset-bundle (generate DAB from Mermaid diagrams)
- databricks-local-notebook (local development)

**For Databricks Data Pipelines:**
- databricks-asset-bundle (pipeline orchestration with serverless)
- databricks-local-notebook (notebook development)
- pytest-test-creator (testing)
- mermaid-diagrams-creator (pipeline visualization)

**For Documentation/Analysis:**

- jira-epic-creator (project planning)

**For Architecture & Planning:**
- mermaid-diagrams-creator (system diagrams)
- databricks-asset-bundle (workflow from diagrams)

### Step 3: Create Project Structure
Execute these actions:

1. **Create directories** according to project structure above
2. **Initialize git repository**
3. **Initialize Python project with uv**:
   ```bash
   uv init --name project-name --python 3.11
   ```
   This creates:
   - `pyproject.toml` with project metadata
   - `.python-version` with Python version
   - `README.md` starter file
4. **Add skills repository as submodule**:
   ```bash
   git submodule add https://github.com/qian-yu-db/custom-claude-skills .claude/skills-repo
   git submodule update --init --recursive
   ```
5. **Create symlinks** for selected skills:
   ```bash
   ln -s ../.claude/skills-repo/[skill-path] .claude/skills/[skill-name]
   ```

## Best Practices

### Python Environment Management
- **Always use uv** for package management (faster, more reliable than pip)
- **Commit `pyproject.toml` and `uv.lock`** to version control
- **Never commit `.venv/`** directory (add to .gitignore)
- **Use `uv run`** to execute scripts in project environment
- **Pin Python version** in `.python-version` file
- **Use `uv sync`** after pulling changes to update environment

### Skill Selection
- Start minimal - add skills as needed
- Consider skill dependencies (e.g., deployment skills need corresponding development skills)
- Review each skill's QUICK_REFERENCE.md for capabilities

### Documentation
- Keep PROJECT_PLAN.md updated as project evolves
- Reference skill documentation for detailed usage
- Document integration points between skills

### Project Organization
- Keep skill-specific code in appropriate directories
- Use clear naming conventions
- Maintain separation between generated and custom code

### Version Control
- Commit .claude/project-context.md with project
- Commit pyproject.toml and uv.lock for reproducibility
- Document skill versions used (via submodule commit)
- Include .gitignore appropriate for tech stack (use uv's defaults)

## Python Environment Management with uv

### Why uv?
- **10-100x faster** than pip for package installation
- **Automatic virtual environment** management
- **Reproducible builds** with uv.lock file
- **Compatible** with all existing Python packages
- **Modern** pyproject.toml-based configuration

### Basic uv Commands

**Install dependencies**:
```bash
uv sync
```

**Add a new package**:
```bash
uv add package-name
```

**Add a dev dependency**:
```bash
uv add --dev pytest
```

**Run commands in project environment**:
```bash
uv run python script.py
uv run pytest
```

**Update dependencies**:
```bash
uv lock --upgrade
uv sync
```

### Project Setup with uv

When bootstrapping a new project:

1. **Initialize project**:
   ```bash
   uv init --name my-project --python 3.11
   ```

2. **Add dependencies from selected skills**:
   ```bash
   # For LangGraph skills
   uv add langchain langgraph langchain-community

   # For Databricks skills
   uv add databricks-sdk databricks-connect

   # For testing
   uv add --dev pytest pytest-cov
   ```

3. **Sync environment**:
   ```bash
   uv sync
   ```

4. **Run project**:
   ```bash
   uv run python src/main.py
   ```

## Troubleshooting

### uv Not Installed
Install uv:
```bash
# macOS/Linux
curl -LsSf https://astral.sh/uv/install.sh | sh

# Windows
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

# Or with pip
pip install uv
```

### Submodule Issues
If skills-repo submodule is not properly initialized:
```bash
git submodule update --init --recursive
```

### Symlink Issues
On Windows, ensure Developer Mode is enabled or use directory junctions:
```bash
mklink /D .claude\skills\skill-name .claude\skills-repo\path\to\skill
```

### Skill Conflicts
If skills have conflicting requirements:
1. Review each skill's SKILL.md for dependencies
2. Use `uv add package==version` to specify compatible versions
3. Check `uv.lock` for resolved dependencies
4. Document resolution in REQUIREMENTS.md

### Python Version Issues
If project requires specific Python version:
```bash
# Set Python version
echo "3.11" > .python-version

# Install specific Python version with uv
uv python install 3.11

# Sync with new version
uv sync
```
